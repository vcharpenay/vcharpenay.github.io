JSON

JSON: JavaScript Object Notation

Une valeur JSON est un objet, un tableau ou une valeur simple.

Type Example
Null null
Boolean true, false
Number 2.4
String "abc"
Object { "a": 2.4, "b": "abc", ... }
Array [ 2.4, "abc", ... ]
Les six types JSON

"That’s it. That is all of JSON." — Douglas Crockford (Crockford, 2008)

Tous les nombres, même les entiers, sont représentés en virgule flottante (double précision).

Les valeurs true et "true" sont différentes (idem pour false et null).

JSON’s design goals were to be minimal, portable, textual, and a subset of JavaScript. The less we need to agree on in order to interoperate, the more easily we can interoperate. — ibid

CSV (comme TSV) est encore plus mimimal et portable.

Pourquoi JSON ?

JSON permet de représenter des données imbriquées.

JSON permet de représenter des données polymorphiques.

JSON permet (aussi) de représenter des données tabulaires.

JSON et programmation

JSON est dérivé de JavaScript.

Un fichier JSON est un ficher JavaScript valide.

JavaScript n'est pas le seul langage dans lequel on peut facilement manipuler des données JSON.

C'est aussi le cas en Python.

Et dans au moins 60 autres langages, d'après le site json.org.

Dans la plupart des langages de programmation, récupérer une valeur dans une structure JSON est trivial : 

f.geometry.coordinates[0] // 4.4038713
f['geometry']['coordinates'][0] # idem

Comment accéder à plusieurs valeurs en une seule requête ?

Par exemple, à travers une expression JSONPath.

On peut par exemple vouloir sélectionner un certain type d'objets GeoJSON dans un tableau.

$[?@.geometry.type == "Point"]

Faites le test sur jsonpath.com.

Voir d'autres exemples issus de la spécification technique.

XML

XML utilise une notation différente de JSON.
Par ailleurs, XML rajoute une distinction entre attributs et éléments.
XPath JSONPath Intended result
/store/book/author $.store.book[*].author the authors of all books in the store
//author $..author all authors
/store/* $.store.* all things in store (some books and a red bicycle)
/store//price $.store..price the price of everything in the store
//book[position()<3] $..book[0,1]
$..book[:2]		 the first two books
//book[isbn] $..book[?(@.isbn)] filter all books with isbn number
//book[price<10] $..book[?(@.price<10)] filter all books cheaper than 10

JSON est nettement plus utilisé qu'XML. Mais...

Le format HTML, spécifiant la structure des pages Web, est un sous-ensemble d'XML.

Plus exactement, XHTML est un sous-ensemble d'XML.

Le format XLS(X) est lui aussi un sous-ensemble d'XML.
