Données ouvertes

Donnée ouverte (déf.) : donnée diffusée de manière structurée selon une méthodologie et une licence ouverte garantissant son libre accès et sa réutilisation par tous, sans restriction technique, juridique ou financière.

Formats de données

Le texte brut est le format le plus simple. Il n'impose aucune structure.

Voir Exemples.

Attention au mode d'encodage des données : UTF, ASCII ou ISO-Western-Europe-* ?

Les formats CSV (comma-separated values) et TSV (tabulation-separated values) sont de loin les plus utilisés. Ils permettent de facilement encoder des tableaux.

Voir Exemples.

Le format JSON (JavaScript Object Notation) est le plus simple à utiliser depuis un programme.

Voir Exemples.

Le format XML (Extended Markup Language) est issu des technologies du Web, il permet de représenter des structures arborescentes.

Voir Exemples.

Les formats RDF (Resource Description Framework) comme Turtle permettent de représenter n'importe quelles données en graphe.

Voir Exemples.

Extraction, vérification, traitement

Quel(s) outil(s) utiliser pour traiter des données hétérogènes aux formats CSV, JSON, XML et autres ?

Excel est si important dans le traitement de données qu'il a son propre sujet sur Reddit (r/excel).

Le concept de feuille de calcul et de macro aura fait son succès.

Depuis, d'autres outils de traitement de données se sont aussi imposés, comme Power BI ou Tableau.

Ces outils facilitent le développement des tableaux de bord pour l'aide à la décision.

Pour des usages avancées d'analyse de données, les outils les plus courant sont des bibliothèques Python :

  • Pandas
  • NumPy
  • Matplotlib
  • (Scikit-learn, PyTorch, etc.)

Publication

La publication de données (ouvertes ou non) se fait essentiellement sur le Web.

Les données peuvent être générées dynamiquement à travers une interface de programmation (API) Web.

Voir par exemple des plates-formes de trafic maritime ou aérien.
