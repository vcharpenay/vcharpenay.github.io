Preliminaries

Activity: period of time during which agents consume and produce artifacts


Project: (partially ordered) set of activitie

Project Management: (efficient) planning and execution of the project


The scope of this course are tools for agile project management.

Agile development is iterative. Many iterations implies many (versioned) artifacts.

Agile development is decentralized. An agile team is self-organized.

International project management is a particular case of decentralized project management.

Pandemic-time management is another.

Agile project development is not limited to software artifacts or source code.


Any (textual) artifact can be produced in an agile fashion.

Other examples:

  • Construction work (design phase)
  • Industrial products (simulation)
  • Technical standards, e.g. by IEC, W3C

Stigmergy

The fundamental concept that most agile development tools adopt for collaboration is stigmergy.


Stigmergy is an indirect form of communication in which agents leaves traces in a shared environment.

Stigmergy in life sciences: termites.


Stigmergy in life sciences: ant colonies.


Stigmergy in social sciences: Wikipedia.

Stigmergy in engineering: Linux.

Rules of stigmergy in agile projects:

  1. Project artifacts are visible to all project members (shared environment)
  2. Artifacts to consume are associated with "requests for actions" (traces)
  3. Artifacts to produce are associated with "change logs" (traces)

Git Basics

First, get a local copy of the code base of a project.


git clone https://gitlab.emse.fr/<username>/<project>.git

Create a new branch of the code base. Each feature or bug fix is associated with a branch.


git checkout -b new-feature

Perform changes to the code and commit them, as in a database transaction.


git add <file1> <file2>...
git commit -m "summary of the changes performed"

When the feature is implemented or the bug fixed (after several commits), return to the master branch and fetch (or pull) the latest remote copy of the code base. 


git checkout master
git pull

Then, merge the created branch back to the master branch and push the merged code base to the remote location. 


git merge new-feature
git push

The Github Flow introduces a slightly more elaborate usage of Git.

The Gitlab Flow encourages an even more elaborate usage of Git.

Large-scale projects may have even more complex workflows with Git.

Code releases are generally numbered according to the semantic versioning guidelines.


v2.4.20

Major version Is backward incompatible with previous versions
Minor version Includes new functionalities
Patch version Includes bug fixes

A release process is generally automated through a production pipeline. Such pipelines are outside the scope of this lecture.

Kanban

The two main agile methods are Scrum and Kanban.


Gitlab integrates functionalities to mix both methods in a single project (what is also called Scrumban).

Kanban emerged as a methodology to manage inventories at Toyota.

A Kanban is a piece of paper that helps visualize the material flow along a supply chain.

The Kanban methodology was then adapted to immaterial "work" flows, through Kanban boards.

Gitlab Basics

A product backlog materializes on Gitlab as a list of issues.


Issues can be organized as Kanban boards through labels and assigned to specific milestones (i.e. sprints).

The free and open-source GNOME project is managed entirely with Gitlab.

The long-term vision of GNOME (multiple releases ahead) is managed as a Kanban board.

Its short-term vision (single release) is managed via Scrum.

Concrete implementation work (individual issue) is again managed with Kanban boards.
