Preliminaries

Activity: period of time during which agents consume and produce artifacts


Project: (partially ordered) set of activitie

Project Management: (efficient) planning and execution of the project


The scope of this course are tools for agile project management.

Agile development is iterative. Many iterations implies many (versioned) artifacts.

Agile development is decentralized. An agile team is self-organized.

International project management is a particular case of decentralized project management.

Pandemic-time management is another.

Agile project development is not limited to software artifacts or source code.


Any (textual) artifact can be produced in an agile fashion.

Other examples:

  • Construction work (design phase)
  • Industrial products (simulation)
  • Technical standards, e.g. by IEC, W3C

Stigmergy

Stigmergy is an indirect form of communication in which an agent leaves traces in the environment that other agents can perceive.

Stigmergy in engineering: Linux.

Single shared environment (code base) with version control.

Traces left: "requests for action" (issues), close to the related artifacts. Same for doc (hence, Javadoc, README)

The concept of stigmergy underlies most agile methods, such as Scrum (traces = backlog) or Kanban (know Trello?)

Combination of both to adapt to scale change: Scrumban.
